The following is an exercise  from Programming Language Concepts by C. Ghezzi and M. Jazayeri.  See page 108.

For the following C like program fragment, describe each stage in the life of the run-time stack until routine beta is called (recursively), by alpha.  In particular, show the dynamic and static links before each routine call.
 
 

    int i = 1, j = 2, k = 3;
    beta( );

    alpha( ){

      int i = 4, l = 5;
      ....
      i += k + l;    //  stack after alpha called from beta but before beta called
      beta( );
      ....
    };

    beta( ){

      int k = 6;
      ....
      i = j + k;  //  stack after beta called from main ...  stack after beta called from alpha
      alpha( );
      ....
    };

    main( ){

      ....  //  stack before beta is called
      beta( );
      ....
    }

General Notes:

A dynamic link is a pointer to the beginning of the caller function's activation record.  Dynamic links are used to determine how many "items" to pop off the stack when a function exits, because an activation record is variable in length.

Static links are pointers to the beginning of the activation record of the unit that statically encloses a given block/function in the program text.  These links are used to access non-local variable.  In this case as in C and C++ there are no nested functions.  In this example, all functions are global in scope and are statically enclosed in the file; therefore, every function's static link points to the global activation record.
