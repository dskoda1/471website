Three Techniques for Functions with Alternatives 

Pattern Matching

> factorialP :: Int -> Int
> factorialP 0 = 1
> factorialP n = n * factorialP(n -1)
  • What happens?  
    If you execute
      ...> factorialP (-3)
      ...> factorialP - 3
    If the code was
      > factorialP n = n * factorialP n -1 

    • Use the type information to avoid defining a factorial function for decimals.
    • Can not constraint type to only positive Int.

Guards

> factorial :: Int -> Int 
> factorial n
>    | n < 0     = error "neg n"
>    | n == 0    = 1
>    | otherwise = n * factorial(n - 1)

    "If" Expression
> factorialE :: Int -> Int
> factorialE n =
>     if n < 0 then error "neg n"
>     else if n == 0 then 1
>          else n * factorialE (n-1) 

Haskell picks the most general type (Type Inference)

  • What is the type of fact1 ?
    • >   fact1 x
    >     | x < 0     = error "neg x"
    >     | x == 0    = 1
    >     | otherwise = x*fact1(x-1)
  • What happens and why?
    • ...> fact1 3.7
  • What happens and is it correct?
    • ...> factorial 13
    1932053504
  • When it should be 6227020800 (= 1932053504 + 2 * 232)
