What is Prolog?
  • Prolog is a ‘typeless’ language with a very simple syntax.
    • The type is bound to the variable at run time.
  • Prolog is a logical programming language.
  • Prolog uses
    • Horn clauses
    • implements resolution
    • using a “depth first” strategy to search the solution space
      • backtracking when a goal fails
    • unification used to instantiate variables
  • Atoms, lists and records are the data structures.

  • A Prolog program is structured like the statement of a Theorem
    • Set of clauses (rules)
    • Set of facts -- database
    • Interactive query

Basic syntax of Edinburgh Prolog (EBNF)

< fact >   ::=     < term >  .

< rule >  ::=    < term >  :-  < terms > .

< query> ::=     < terms > .

< term > ::=   < number  >   |   < atom >
                     |   <variable>   |   < atom > (  < terms  > )
<eval> ::=   <variable> is <expr> | <number> is <expr>

< terms > ::=   < term  >   |    < term  > ,  < terms >
                        |  <eval> | <eval>, <terms>
<expr> ::= <number> | <variable> | <expr> <op> <expr>

Syntax

  • CASE sensitive
  • Integers: base 10
  • atoms: (constants)
    • atoms are a series of letters, digits, and # $ & * + - / \ > = < ? @ ^ ~
      • [ ] { } , have special meaning
      • "/*",   used for comments
    • user defined, supplied
    • name does not start with upper case letters: john,  student2
  • Variables
    • begin with upper case: Who, X
    • ‘_’ can be used in place of variable name
  • Structures/records/tuples/relationship/functors
    • student(ali,  freshman, 194).
  • Lists
    • [x, y, Z ]
    • [ Head | Tail ]
      • syntactic sugar for  . ( Head, Tail )
  • [ ]

Running a simple program.

family.pl
In the swipl window:
1?-  consult ( ‘family.pl’ ).
family.pl compiled, 0.00 sec, 828 bytes
  • Comments are either bound by “/*”,”*/” or any characters following the “%”.
  • Structures are just relations.
    • They are not functions.
    • There are no inputs or outputs for the variables of the structures.
  • The swipl documentation of the built-in predicates does indicate how the variables should be used.
      • pred(+var1, -var2, +var3).
        + indicates input variable
        - indicates output variable
  • You can also consult (load) a file with a “pl” extension by
      • ?- [family].

Sample execution:
  • In Prolog the sequence of the facts and rules is the order the facts and rules are searched through
  • Variation from pure logic model
      •  
    2 ?- father( X, Y ).

    X = mike   /* BLUE represents computer output */
    Y = ann ;  /* User types ‘;’ to continue searching the data */

    X = mike
