    The Ackermann Number N of the compiler as the largest N for which
              ack (3,N)
    gives an answer without a stack overflow.  In earlier decades a variation had been used as one of the benchmarking algorithms. Here (The Computer Language Benchmarks Game) are current algorithms used for benchmarking.  (There are several different variants of this function.  All are called Ackermann's functions and their values do NOT have to agree!)   
ack( m,n ) = n + 1
 if m = 0
ack( m,n ) = ack(m - 1, 1)
 if n = 0 and m > 0 
ack( m,n ) = ack( m-1, ack( m, n-1 ) )       if n >0 and m > 0
