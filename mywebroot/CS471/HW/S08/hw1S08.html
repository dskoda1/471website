[Rosen,Discrete Mathematics and Its Applications 5th ed., pg 273 #48] "Consider the following inductive definition of a version of Ackermann's function.  This function was named after Wilhelm Ackermann, a German mathematician who was a student of the great mathematician David Hilbert.  Ackermann's function plays an important role in the theory of recursive functions and in the study of complexity of certain algorithms involving set unions."  In the early days of  imperative languages, Ackermann's function was used to measure the recursion capability of a compiler.

    The Ackermann Number N of the compiler as the largest N for which
              ack (3,N)
    gives an answer without a stack overflow.  Recently, a variation has been used as one of the benchmarks for the Computer Language Shootout.  (There are several different variants of this function.  All are called Ackermann's functions and their values do NOT have to agree!)   

ack( m,n ) = 2 * n if m = 0
ack( m,n ) =  0 if n = 0 and m > 0 
ack( m,n ) = 2if n = 1 and m > 0 
ack( m,n ) = ack( m-1, ack( m, n-1 ) )       if n >1 and m > 0
