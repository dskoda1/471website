    1. [very easy]Write a relation,  mkTuple( X, Y, P) that succeeds if P is a tuple (i.e. a pair) whose first element is X and second element is Y.
       ?- mkTuple(a,b,P).
       P = (a,b)
       yes
       ?- mkTuple(X,Y,(1,2)).
       X = 1
       Y = 2
       yes
       
    2. [very easy] Define the relation,  mkZip(Z,X,Y),where Z is unified with zip(X,Y).  This is really easy if you use pattern matching and unification.

       ?- mkZip(Z,a,b).
       Z = zip(a,b)
       yes
       ?- mkZip(zip(1,2),X,Y).
       X = 1
       Y = 2
       yes
    3. [easy] Write a predicate, addV(V,Lst,RLst),  such that every element in R is V more than the corresponding element in Lst. i.e.

         ?-addV(2,[1,2,4],X).
         X = [3,4,6]
         yes
         ?- addV(3,X,[3,4]).
         You will get an error...Arguments are not sufficiently instantiated.
