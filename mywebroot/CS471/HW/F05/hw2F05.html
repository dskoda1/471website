[Rosen 211#36] Consider the following inductive definition of a version of Ackermann's function.  This function was named after Wilhelm Ackermann, a German mathematician who was a student of the great mathematician David Hilbert.  Ackermann's function plays an important role in the therory of recursive functions and in the study of complexity of certain algorithms involving set unions.  In the early days of  imperative languages, the Ackermann function was used to measure the recursion capability of a compiler. The Ackermann Number N of the compiler as the largest N for which
              ack (3,N)
    gives an answer without a stack overflow. (There are several different variants of this function.  All are called Ackermann's functions.)  Later we will program this function.  
    ack( m,n ) = 2n     if m = 0
    ack( m,n ) =  0     if m >=1 and n = 0 
    ack( m,n ) = 2     if m >=1 and n = 1 
    ack( m,n ) = ack( m-1, ack( m, n-1 ) )     if m >=1 and n >=2

    1. [1pt] ack(1,0) = ______
