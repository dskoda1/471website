Implement a Haskell modules for complex numbers.
The purpose of this exercise is
  • to use Haskell modules for separate compilation and data encapsulation
  • to create an instance (a type) of class Num
A module in Haskell has a name and contains a collections of definitions.  The name of the modules begins with an upper case Letter and is the same as the filename.  All top level definitions of a module are exported by default.  Haskell's modules has features for hiding and importing but we will not used them in this assignment.

Haskell provides a mechanism for types in classes can be created automatically. This is done in the type data type declaration:

> data (RealFloat a)=> Complex a =  Complex a a
>                   deriving (Eq,Read,Show)

This works because the natural definitions of each class for Complex follows from the definition of RealFloat. For example:
 (Complex a b) == (Complex a' b') = (a == a') && (b == b')

The Complex data type should be in the Class Num.
Below is the definition of Num from the standard library specification. 

class  (Eq a, Show a) => Num a  where
    (+), (-), (*)    :: a -> a -> a
    negate           :: a -> a
    abs, signum      :: a -> a
    fromInteger      :: Integer -> a

        -- Minimal complete definition:
        --      All, except negate or (-)
    x - y            =  x + negate y
    negate x         =  0 - x

Complex.lhs is provided to help you get started.  It also has some examples of using this class.

Challenge:
Implement the Complex data type to be in the Fractional Class.  Below is the definition of Fractional from the library specification.

class  (Num a) => Fractional a  where
    (/)              :: a -> a -> a
    recip            :: a -> a
    fromRational     :: Rational -> a

        -- Minimal complete definition:
        --      fromRational and (recip or (/))
    recip x          =  1 / x
    x / y            =  x * recip y
