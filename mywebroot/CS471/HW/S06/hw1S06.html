[Rosen 211#36] "Consider the following inductive definition of a version of Ackermann's function.  This function was named after Wilhelm Ackermann, a German mathematician who was a student of the great mathematician David Hilbert.  Ackermann's function plays an important role in the theory of recursive functions and in the study of complexity of certain algorithms involving set unions.  In the early days of  imperative languages, the Ackermann function was used to measure the recursion capability of a compiler. The Ackermann Number N of the compiler as the largest N for which
              ack (3,N)
    gives an answer without a stack overflow. " (There are several different variants of this function.  All are called Ackermann's functions.)  Later we will program this function.  
[Mathworld.wolfram.com] Ackermann's function is also of interest because it is an example of a well defined total function which is computable but not primitive recursive.  

Below is one variation of the definition of the function.

ack( m,n ) = n+1      if m = 0
ack( m,n ) =  ack( m-1, 1)      if n = 0 and m > 0 
ack( m,n ) = ack( m-1, ack( m, n-1 ) )        n > 0 and m > 0)
